1. НАЗНАЧЕНИЕ ПРОГРАММНОГО МОДУЛЯ CAD
Программный модуль CAD разработан на базе интегрированной инженерной программной платформы и программно-математического ядра трехмерного моделирования «RGK».
Функциональное назначение программного модуля CAD:
1) создание и редактирование геометрической модели объектов проектирования, включая 3D- и 2D-представления;
2) задание атрибутивной информации, связанной с точностными, размерными, технологическими параметрами объектов проектирования (PMI);
3) оформление графической документации в соответствии с требованиями стандартов (ЕСКД);
4) визуализация объектов проектирования с различными параметрами отображения.
2. УСЛОВИЯ ВЫПОЛНЕНИЯ ПРОГРАММНОГО МОДУЛЯ CAD
Достаточными условиями выполнения программного модуля CAD являются системные требования:
5) ОС – Microsoft Windows версии не ниже 10, Astra Linux версии 1.7 и выше;
6) процессор Intel Core i3 или выше;
7) видеокарта NVIDIA или AMD с памятью 1ГБ и выше;
8) ОЗУ 2 ГБ и более;
9) объем свободного пространства на жестком диске, необходимого для установки – 1 ГБ и более.

3.3.3. Сохранение документов
Команда «Сохранить»   выполняет сохранение текущего активного документа. Если документ был создан ранее и находился на редактировании, то команда выполнит сохранение этого документа, а если работа происходила в новом документе, то система предложит выбрать директорию для сохранения и указать имя и формат файла.
Команда «Сохранить как»   вызывает стандартный диалог сохранения активного документа в файл с возможностью выбора директории сохранения, задания имени и типа файла.
При необходимости система может автоматически сохранять документ с заданным интервалом по времени. Автосохранение позволяет минимизировать риск потери данных в результате незапланированного закрытия системы. Активации функции автоматического сохранения осуществляется в диалоге параметров автосохранения (рис. Рисунок).
Параметры автосохранения
 
Рисунок

Поле «Интервал сохранения» задает интервал автоматического сохранения в минутах.

3.3.5.2. Атрибуты документа
Атрибутом документа называется переменная, помеченная флагом «Системная». Атрибуты документа, как и переменные, могут быть текстовыми и вещественными. 
Атрибуты документа необходимы для управления метаданными документа и для решения задач управления электронной структурой изделий (ЭСИ) при организации коллективной работы. Значения атрибутов подлежат синхронизации с параметрами объектов электронной структуры изделия. 
Атрибуты документа заранее созданы во всех новых документах в соответствии с прототипами, на основе которых создаются эти документы.
Отображение списка атрибутов активного документа осуществляется из меню «Файл», при этом активируется соответствующая вкладка на панели «Лента» (рис. Рисунок).
Окно атрибутов документа
 
Рисунок 
По умолчанию в списке атрибутов отображаются только колонки «Наименование» и «Значение». В контекстном меню списка можно активировать отображение колонок «Выражение» и «Переменная». Каждому атрибуту присвоена одноимённая системная переменная. Таким образом, присваивая переменным статус управляющей, обеспечивается возможность связывания атрибутов с разными конфигурациями, а также с другими документами.
Команда «Добавить» предназначена для создания нового атрибута. После запуска команды откроется окно со списком системных атрибутов, которые могут быть добавлены в документ. По умолчанию во всех документах создан полный набор атрибутов, поэтому при попытке добавления нового список будет пустым. Если же какой-то атрибут был ранее удалён, то при добавлении его имя отобразится в списке.
Команда «Удалить» удаляет выбранный атрибут из списка.
Команда «Копировать в буфер»   копирует выбранные атрибуты в буфер обмена.
Команда «Вставить из буфера»   вставляет скопированные атрибуты. Атрибуты могут быть скопированы из одного документа и вставлены в другой.
Команда «Сохранить в Файл»  экспортирует атрибуты документа во внешний XML-файл. После запуска команды откроется проводник операционной системы, где необходимо задать имя создаваемого файла и выбрать папку, в которую он будет сохранён.
Команда «Считать из файла»  импортирует атрибуты в текущий документ из внешнего файла. После запуска команды откроется проводник операционной системы, где необходимо выбрать XML-файл, из которого будут считаны атрибуты.

Информация об истории документа может быть получена из окна «История документа»   (рис. Рисунок), которое вызывается из меню «Файл».

4.1. Окна программного модуля CAD
Программный модуль CAD поддерживает многодокументный многооконный интерфейс (MDI). Это позволяет создавать произвольное число главных окон, каждое из которых может размещаться произвольно, в том числе на разных мониторах.
Главные окна являются равнозначными. Открытие или закрытие любого из главных окон не меняет функциональность работы программного модуля CAD в текущем сеансе. Программный модуль CAD завершает работу при закрытии последнего главного окна.
Открытие нового главного окна осуществляется при помощи команды «Новое главное окно», расположенной на вкладке «Вид» инструментальной панели «Лента» (группа «Окна»). Вновь открытому главному окну присваивается очередной порядковый номер, указанный в квадратных скобках в области его заголовка (рис. Рисунок).

Многодокументный интерфейс обеспечивает возможность открытия нескольких документов, каждый из которых может редактироваться отдельно. Вне зависимости от того, в каких главных окнах открыты рабочие окна документов, происходит полноценная синхронизация изменений объектов.
4.1.1. Рабочие окна
Рабочее графическое окно является одним из основных элементов интерфейса программного модуля CAD для работы с объектами модели, использующееся для их визуализации в различных представлениях с последующим выполнением различных операций над ними. Основными типами рабочих окон являются:
1) «3Dвид» – предназначен для отображения сцены с 3Dобъектами модели;
2) «2Dвид» – предназначен для отображения 2Dобъектов модели;
3) «КЭ-сетка» – предназначена для отображения расчётной сетки конечных элементов;
4) «Монитор задач» – предназначен для отображения состояния работ различных бизнес-процессов;
5) «Редактор графиков» – предназначен для работы с графическими зависимостями;
6) «Редактор конфигураций» – предназначен для создания конфигураций модели;
7) «Редактор таблиц» – предназначен для создания и редактирования таблиц;
8) «Редактор переменных» – предназначен для работы с переменными (специальными элементами модели, позволяющими задавать различные виды негеометрических взаимосвязей между объектами моделей).
Для каждого документа может быть открыто одно или несколько рабочих окон.
Рабочие окна всех документов, открытых в текущем сеансе работы, представляются в виде вкладок на панели, расположенной под инструментальной панелью «Лента» (рис. Рисунок).

Группа «Цвета состояний объекта» содержат параметры для задания цвета объектов при динамической подсветке, а также при явном выборе курсором в рабочем окне.
Группа «Эскиз» включает:
64) «Автоматический подбор угла в плоскости эскиза» – параметр, отвечающий за автоматический подбор значения угла поворота плоскости эскиза;
65) «Автоматический разворот активной плоскости эскиза» – параметр, отвечающий за автоматический разворот 3Dсцены плоскостью создаваемого эскиза параллельно экрану;

Для применения изменений параметров установок необходимо перезагрузить программу, о чём свидетельствует соответствующее уведомление, отображающееся при закрытии окна установок (рис. Рисунок). Немедленное применение изменений осуществляется при установке флага «Применять установки без перезапуска».

5. МОДЕЛИРОВАНИЕ
5.1. Сервисные функции
5.1.1. Регистрация действий по изменению модели
Программный модуль CAD регистрирует все действия по изменению состава, параметров и связей объектов модели документа. Это обеспечивает возможность ведения истории документа и протоколирования изменений объектов с целью обеспечения функций по их отмене и возврату.
Информация об истории документа может быть получена при вызове диалогового окна «История документа» из выпадающего меню «Файл».
Функции отмены и возврата действий по изменению объектов модели расположены на панели быстрого доступа.
Кнопка «Отмена»   производит отмену последнего действия. Каждое нажатие кнопки отмены соответствует одной транзакции по изменению состава действий текущего документа.
Кнопка «Повтор»   выполняет возврат отменённого действия. Каждое нажатие кнопки повтора соответствует одной транзакции по изменению состава действий текущего документа.
Функции отмены и повтора имеют раздельный учёт последовательности действий для разных документов в одном сеансе работы, поэтому их использование в активном документе не применяется к другому документу.
5.1.2. Регенерация модели
В ходе создания модели между объектами могут возникать всевозможные связи, которые будут иметь различный характер. В зависимости от последовательности действий по изменению состава, параметров и связей между объектами формируется иерархическая структура зависимостей между объектами.
Окно «Зависимости» содержит информацию о взаимосвязях выбранного объекта модели:
67) с родительскими объектами (от которых зависит выбранный объект);
68) с дочерними объектами (которые зависят от выбранного объекта).
В свою очередь, каждый родительский объект может зависеть от других объектов, а каждый дочерний может иметь зависимые. О наличие зависимостей свидетельствует пиктограмма   рядом с объектом, при нажатии которой раскрывается цепочка зависимостей (рис. Рисунок). Повторное нажатие пиктограммы сворачивает цепочку зависимостей.

Все объекты «Навигатора модели» могут быть переименованы. Для этого следует вызвать контекстное меню объекта, в нём выбрать пункт «Свойства» и в поле параметра «Имя» ввести требуемое наименование. До подтверждения изменений параметр «Имя» будет помечен красной звёздочкой.
Для поиска объекта в навигаторе следует ввести его имя в верхней строке диалога и нажать <Enter>. Если объект (объекты) с таким именем существует, то группа, содержащая его, раскроется, и объект будет подсвечен в навигаторе. Колонка   предназначена для подавления объектов. Подавленный объект исключается из пересчёта модели и не отображается в 3Dсцене.

Любая операция или объект могут быть отредактированы из «Навигатора модели». Для этого её необходимо выбрать, открыть контекстное меню и выбрать пункт «Редактировать». В результате запустится диалог соответствующей команды, в котором можно внести изменения в операцию.
Изменения, которые вносятся в документ, отражаются в «Навигатором модели». Например, если в некоторое тело была добавлена операция, то данная операция будет включена в состав тела в дереве модели.
Если в ходе создания либо редактирования модели возникают ошибки, проблемные объекты и операции маркируются восклицательным знаком, а в окне диагностики появляются соответствующие сообщения.
Можно менять порядок операций в дереве модели при необходимости. Для этого необходимо выделить операцию в папке «История операций» и, удерживая левую кнопку мыши, переместить ее выше или ниже в порядке построений. Если перемещение на выбранный этап построения возможен, рядом с названием перемещаемой операции будет отображаться ее иконка, а ее будущее «место» в дереве будет отмечено черной линией (рис. Рисунок). Если при перемещении рядом с названием операции отображается красная пиктограмма, перемещение на данный этап построения невозможен.

«Точка» отката отмечается в истории операций пиктограммой  . По умолчанию она располагается в конце истории, а новые операции добавляются до неё. Эта пиктограмма также позволяет откатить модель до нужного этапа. Достаточно «перетянуть» ее в нужное место в истории операций.

Твёрдые и поверхностные тела, операции, точки и кривые могут быть скопированы через буфер обмена, и их можно вставить в тот же документ или в другой. Ассоциативная связь при этом не сохранится. Для этого в контекстном меню объектов в «Навигатором модели» и 3D-сцене представлены команды «Копировать»    и «Вставить»  . При вставке объект-копия сохранит свое положение относительно абсолютной СК документа.
Объект или папка «Навигатора модели» для удобства могут быть открыты в отдельном окне. Для этого в их контекстном меню нужно вызвать команду «Открыть в отдельном окне».
5.1.6. Свойства объектов
Окно «Свойства» содержит информацию обо всех свойствах выбранного объекта, в том числе и о свойствах материала.

5.1.8. Управление объектами через слои
Слой – параметр каждого геометрического объекта, который определяет его принадлежность какой-либо группе элементов модели.
Вызов «Навигатора слоёв»   осуществляется из панели «Лента» (вкладка «Вид», группа «Инструменты» (рис. Рисунок)).
Окно «Слои»
 
Рисунок 
Слоям и группам слоёв могут быть назначены пользовательские имена. Для переименования слоя дважды кликните ЛКМ по его имени и введите в поле новое имя слоя.
«Навигатор слоёв» содержит список слоёв документа и набор команд для управления ими.
Команда   «Добавить новый слой» позволяет создать новый слой. При нажатии кнопки создается слой, которому автоматически присваивается название, которое допускается изменить.
Команда   «Установить текущий слой» делает выбранный слой текущим: на него будут помещаться вновь создаваемые объекты модели.
Команда   «Удалить слой/группу слоёв» удаляет выбранный слой или группу слоёв.
Команда   «Перенести объекты на слой» позволяет назначить предварительно выбранным объектам модели выбранный слой.
Команда   «Добавить стиль слоя» назначает определенный стиль объектам на выбранном слое. Существуют следующие варианты выбора стилей: «Материал», «Стиль линии», «Стиль заполнения», «Стиль 2Dточки», «Стиль 3Dточки», «Стиль РП», «Стиль ЛСК», «Стиль текста», «Параметры сетки». При выборе одного из перечисленных вариантов происходит заимствование его стиля и присвоение объектам выбранного слоя. Для стиля слоя существует набор стандартных команд: «Свойства», «Зависимости», «Редактировать», «Удалить».
Команда   «Удалить стиль слоя» позволяет удалить стиль, созданный ранее командой «Добавить новый стиль в текущий слой». При удалении одного из заимствованных стилей программный модуль CAD запросит подтверждение, а также предложит удалить зависимые объекты.
Команда   «Свойства слоя/группы слоёв» открывает окно свойств с параметрами выбранного слоя или группы слоёв. Доступно свойство «Цвет пометки», назначающее выбранный цвет объектам слоя.
Команда   «Создать группу слоёв» создает группу, которая позволяет комплесно управлять входящими в неё слоями: атрибуты, назначенные группе, распространяются на все входящие в неё слои (рис. Рисунок).

5.1.9. Управление приоритетом объектов
По умолчанию очерёдность отображения перекрывающихся объектов определяется автоматически в зависимости от их типов. При этом программный модуль CAD предусматривает возможность ручного управления очерёдности за счёт назначения приоритетов.
Под приоритетом следует понимать порядковый номер прорисовки 2Dобъекта. Таким образом объекты с более высоким значением приоритета перекрывают объекты с более низким значением. Текущее значение приоритета объекта отображается в окне свойств. По умолчанию значение приоритета равно 0.
Для управления приоритетом объектов предусмотрена команда «Порядок объектов». Данная команда предусматривает шесть режимов:
91) «На передний/задний план» – текущий объект отображается над/под остальными перекрывающимися объектами (рис. Рисунок). Объекту назначается максимальное либо минимальное значение приоритета соответственно;

92) «Переместить вперёд/назад» – текущий объект перемещается на один уровень выше/ниже (рис. Рисунок) Значение приоритета увеличивается либо уменьшается на единицу соответственно;

5.1.10. Сцены
Команда «Создать сцену» предназначена для создания в модели новой 3D-сцены.
Команда запускается из контекстного меню, вызванного на объекте «Сцена» в «Навигатором модели». После запуска откроется диалог параметров (рис. Рисунок).
Диалоговое окно создания сцены
 
Рисунок 
Параметр «Базовая сцена» предназначен для выбора сцены, на основе которой будет создана новая сцена.
Блоки «Исключаемые объекты» и «Добавляемые объекты» предназначены для выбора объектов (тел и компонентов сборки), которые должны быть исключены из сцены или добавлены в неё.
Блок «Модифицируемые объекты» предназначен для выбора тел и компонентов, которые должны быть трансформированы в сцене. Для задания трансформации справа от поля выбора объектов находится пиктограмма одноименной команды.
В блоке «Параметры» содержатся настройки визуализации новой сцены.
Флаг «Активировать сцену» позволяет сделать созданную сцену активной после создания. 
5.1.11. Управление представлениями
Управление представлениями рабочего окна предполагает следующие манипуляции:
1) вращение 3Dпредставления (сцены) относительно некоторой точки, осуществляющееся за счёт перемещения курсора в пределах рабочего графического окна при захвате ПКМ (если в момент захвата сцены под курсором находится геометрический объект, тогда центром вращения сцены будет пересечение луча, проходящего через точку захвата перпендикулярно плоскости экрана, с данным объектом. Такая точка будет маркирована пиктограммой  . В противном случае в качестве центра вращения будет выступать центр сцены);
94) перемещение представления (2D и 3D) в плоскости экрана, осуществляющееся за счёт перемещения курсора в пределах рабочего графического окна при захвате СКМ;
95) масштабирование представления (2D и 3D), осуществляющееся СКМ, при этом масштабирование происходит относительно текущего положения курсора в рабочем графическом окне. Изменение направления прокрутки колеса мыши осуществляется за счёт параметра «Обратное направление колеса мыши» в окне установок программного модуля CAD.
Для управления 3Dсценой в графическом окне дополнительно предусмотрен специальный инструмент «Видовой куб», расположенный в правом верхнем углу рабочего окна (рис. Рисунок).

Пиктограмма   позволяет сориентировать модель согласно базовому положению. Пиктограммы   и   позволяют поворачивать сцену в плоскости рабочего окна по часовой и против часовой стрелки соответственно.
